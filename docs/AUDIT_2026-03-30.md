### Note on C8/C9 (provider prefix stripping)
This is a latent bug rather than an active one: it only triggers if a model is stored in config with a `"provider/model"` prefix, which currently only happens for `minimax-portal`. Other providers (Anthropic, OpenAI, Azure) are typically configured without the prefix so the bug is dormant — but will activate the moment a second non-MiniMax provider uses prefixed model names. Fix proactively.
