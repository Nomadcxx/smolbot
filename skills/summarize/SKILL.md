Use this skill to create concise summaries of content.

## When to Use

- Summarize lengthy conversation history
- Create executive summaries of documents
- Distill key points from code reviews
- Generate brief overviews of complex topics
- Create meeting summaries from transcripts

## Summary Principles

1. **Capture essential meaning** - Preserve the core message
2. **Use selective detail** - Keep important specifics, discard noise
3. **Maintain accuracy** - Don't distort or oversimplify
4. **Preserve structure** - Group related points together
5. **Be concise** - Aim for 10-20% of original length

## Summary Types

### Conversation Summary
For conversation history compression:
- Key decisions made
- Outstanding questions or issues
- Tasks completed or in progress
- Important context for future work

### Document Summary
For document distillation:
- Main purpose or goal
- Key sections and their content
- Important findings or data
- Conclusions and recommendations

### Code Summary
For code understanding:
- What the code does
- Key functions and their purpose
- Important data structures
- Dependencies and relationships

## Output Format

- Brief heading stating the topic
- Bullet points for key information
- Any critical details that must not be lost
- Format that's easy to scan and reference
